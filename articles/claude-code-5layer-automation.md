## はじめに

毎朝起きたら、メールボックスにこんなものが届いている。

- 入札・公募案件の調査結果
- セキュリティスキャン結果
- 競合の最新動向
- Note記事の下書き3本
- X投稿のドラフト6本
- 戦略レポート

全部AIが深夜3時から勝手にやってくれたものだ。さらに、PCを5分放置するだけでタスクキューが自動消化される「アイドルデーモン」も常駐している。

自分は一人で事業をやっていて、開発もマーケもリサーチも全部自分でやる必要がある。そこでClaude Codeの`claude -p`（プログラマブルモード）を使って、5層構成の自動化環境を作った。寝ている間も、昼休みも、会議中も——PCがある限りAIが働き続ける。

この記事では、その全体構成と実装のポイントを共有する。

## 全体像：5層の自動化アーキテクチャ

```
 3:00  ┌─────────────────────────────┐
       │  夜間ワーカー（night_worker） │  ← 本記事のメイン
       │  並列11タスク → 戦略統合      │
 6:45  └─────────────────────────────┘
 7:00  ┌─────────────────────────────┐
       │  チャンネル収集                │  ← ニュース＋SNSドラフト＋メール
 7:30  └─────────────────────────────┘
12:00  ┌─────────────────────────────┐
       │  ランチワーカー               │  ← ヘルスチェック＋午後タスク提案
       └─────────────────────────────┘
17:00  ┌─────────────────────────────┐
       │  イブニングワーカー            │  ← 振り返り＋翌日準備
       └─────────────────────────────┘
常駐   ┌─────────────────────────────┐
       │  アイドルデーモン              │  ← PC放置5分→タスク自動消化
       └─────────────────────────────┘
```

launchd（macOSのcronみたいなもの）で時刻起動し、各ワーカーが`claude -p`を並列に走らせる。

## 夜間ワーカーの仕組み

一番のメインがこの夜間ワーカー。6つのフェーズで構成されている。

### Phase 1：データ収集＋コンテンツ生成（並列11タスク）

```bash
# 入札リサーチ、セキュリティスキャン、Azure調査、競合調査...
# すべてバックグラウンドで同時実行

(
  claude -p "入札・公募案件を調査して..." \
    --allowedTools "WebSearch,WebFetch,Read,Write" &
)
P1_PIDS+=($!)

# 他のタスクも同様に並列起動...

# 全タスクの完了を待つ（wait_namedは自前のwait関数）
wait_named P1_NAMES P1_PIDS
```

実際のスクリプトでは11タスク分あるが、パターンは全て同じ。ポイントは`--allowedTools`でツールを絞ること。WebSearchが不要なタスクにはRead,Writeだけ渡す。これでトークン消費を抑えつつ、安全性も確保できる。

各タスクにはタイムアウト（10分）を設定している。1つのタスクが詰まっても全体が止まらない。

```bash
CLAUDE_TIMEOUT=600
run_claude() {
  gtimeout "$CLAUDE_TIMEOUT" claude -p "$@"
}
```

### Phase 2：状況評価＋ブリーフィング統合

Phase 1の出力ファイルを全部読ませて、1つのレポートにまとめる。

```bash
claude -p "Phase 1で収集した情報（入札、セキュリティ、Azure、競合、JTB動向...）と
前日の戦略レポートを読んで、今日の状況評価レポートを作成してください。" \
  --allowedTools "Read,Write,Glob"
```

「前日の戦略レポート」を入力に含めるのがミソ。日々の分析が蓄積されていく。

### Phase 3〜4：戦略深堀り＋統合

Phase 2のレポートをベースに、3つの視点（事業開発・プロダクト・パートナー戦略）で並列分析。それを最終統合する。

### Phase 5：モーニングレポート

全フェーズの成果を1つの「今日の朝に読むべきレポート」にまとめてメール送信。起きたらメールを読むだけ。

## チャンネルエンジン：7時起動のニュース収集レイヤー

夜間ワーカーとは別に、朝7時にはチャンネルごとのニュース収集が走る。

分野ごとに`config.yml`を1つ書くだけで、ニュース収集→SNSドラフト→メール送信が自動化される。

```yaml

  name: "AI・テック速報"
  id: ai
```

## アイドルデーモン：PCを放置するだけでタスクが消化される

```bash
get_idle_seconds() {
  # macOSのHIDIdleTimeでキーボード/マウスの無操作時間を取得
  ioreg -c IOHIDSystem | awk '/HIDIdleTime/ {print int($NF/1000000000); exit}'
}
```

5分間操作がなかったら、`.queue/pending/`にあるタスクを自動で`claude -p`実行。

タスクの投入はこれだけ：

```bash
bash scripts/queue_task.sh -n "市場調査" "○○業界の市場規模を調べて"
```

昼休み、会議中、寝る前——PCから離れた時間が全部「AIの作業時間」になる。

## 並列エージェント：YAMLレシピで複数AIを同時実行

```yaml
# scripts/recipes/strategy_report/agents.yml
agents:
  - name: market-analyst
    prompt: "市場分析を実施..."
  - name: tech-researcher
    prompt: "技術トレンドを調査..."
  - name: financial-analyst
    prompt: "財務シミュレーション..."

synthesize_prompt: "3つのレポートを統合して..."
```

これで3つのClaude -pが同時に走り、最後に統合レポートが出力される。

## 改善ループ：コンテンツを自動でブラッシュアップ

生成されたNote記事や戦略レポートを、3人の仮想レビュアーが並列でレビューし、そのフィードバックを統合して改善版を生成する。これを収束するまで繰り返す。

```bash
bash scripts/improvement_loop.sh --preset note -i article.md
```

レビュー基準もプリセットで用意してあるので、コマンド1つで品質が上がる。

## インフラのポイント

### macOSで確実に動かす

macOSはスリープすると全部止まる。`pmset`でスケジュール起床し、`caffeinate`でスリープを防止する。

```bash
# 2:55に自動起床
sudo pmset repeat wakeorpoweron MTWRFSU 02:55:00

# 夜間ワーカー内でスリープ防止
caffeinate -u -s -t 13500 &  # 3時間45分
```

### エラー耐性

- lockファイルで二重起動防止
- trapでcleanup確保
- 各タスクにタイムアウト
- 1つ失敗しても他は継続

```bash
cleanup() {
  [ -n "$CAFFEINATE_PID" ] && kill "$CAFFEINATE_PID" 2>/dev/null
  rm -f "$LOCK_FILE"
}
trap cleanup EXIT
```

### claude -pの入れ子実行

Claude Codeの中から`claude -p`を呼ぶには、環境変数`CLAUDECODE`をunsetする必要がある。

```bash
unset CLAUDECODE 2>/dev/null || true
```

これをスクリプトの先頭に書いておく。環境によっては追加の設定が必要なケースもあるが、大半はこれで動く。

## 運用して分かったこと

### 実際にかかるコスト

Claude Pro Max（月額$200）で`claude -p`は追加料金なし。夜間ワーカーの11並列タスクで約30〜50分かかるが、寝ている間なので気にならない。

### 精度について

Webリサーチ系（入札調査、競合調査）は7-8割の精度。たまにソースが古かったり、存在しないURLを参照したりする。ただ「ゼロから自分で調べる」のと比べたら圧倒的に早い。あくまで「下調べを代行してもらう」スタンスで使うのがコツ。

### 蓄積が価値になる

毎日の戦略レポートが積み上がると、トレンドの変化が見えてくる。先週と今週で競合の動きがどう変わったか、入札案件の傾向はどうか——こういう「定点観測」は人間がやると続かない。AIなら毎日淡々とやってくれる。

## まとめ

- Claude Codeの`claude -p`とlaunchdで、24時間稼働のAIアシスタント環境が作れる
- 並列実行＋パイプライン型設計で、1人でも組織的な情報収集・分析が回せる
- macOSではpmset + caffeinateでスリープ問題を解決
- 精度は完璧じゃないが、「下調べの代行」として十分実用的

ソロで事業をやっていて「情報収集と分析に時間が取られすぎる」と感じている人には、この仕組みはかなり効く。
